--- notes: | Needs arise on occasion to add soft breaks with the `br` tag. Since such a break is enforced across all devices by default this can help some sizes but hurt others. We've set up a few helper classes that allow for more responsive use of the br tag: | Class | < 365 | < 760 | < 1020 | 1020+ | |:--------------------------|:------|:------|:-------|:------| | `.br--sm-smartphone-only` | YES* | NO | NO | NO | | `.br--smartphone-only` | NO | YES | NO | NO | | `.br--tablet-only` | NO | NO | YES | NO | | `.br--desktop-only` | NO | NO | NO | YES | | `.br--not-sm-smartphone` | NO** | yes | yes | yes | | `.br--not-smartphone` | yes | NO | yes | yes | | `.br--not-tablet` | yes | yes | NO | yes | | `.br--not-desktop` | yes | yes | yes | NO | \* YES vs. NO here means it is enabled for YES *and restricted* at NO \*\* yes vs. NO here means it is restricted for NO but will be passively enabled at yes unless otherwise restriced by layering several `--not-` modifiers together. ---
